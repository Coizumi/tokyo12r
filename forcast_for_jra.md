# TOKYO12R JRA Forecast Logic

対象実装:

- `scripts/jra_site_updater.py`
- `scripts/jra_feature_pipeline.py`

この文書はTOKYO12Rの予想ロジックのみを定義する。インフラ、更新スケジュール、デプロイ、VPS、Cloudflare、systemd、運用手順は `specs.md` に記載する。

## ロジック概要

TOKYO12Rは、JRA公式出走表から取得した出走馬情報と近走テキストを使い、各レース最大5頭に以下の印を付ける。

```text
◎ ○ ▲ △ ☆
```

現行ロジックは機械学習モデルではなく、決定的な指数型ヒューリスティックである。乱数は使わない。

現行ロジックで使う主な評価軸は以下。

- 近走と実績による基礎スコア
- 持ちタイム指数
- 末脚指数
- 先行力指数
- 父および母の父による血統レース条件適性

現在の印決定では、現行オッズ、斤量、騎手、性齢、天候、馬場状態、調教、厩舎コメント、新聞印は使わない。人気順位は表示用に保持し、近走テキスト内の過去人気だけを基礎スコアの一部として使う。

## 入力として使う情報

`jra_site_updater.py` はJRA公式出走表HTMLから、各馬について以下を内部情報として読む。

- 馬番
- 馬名
- 現行人気順位
- 性齢
- 騎手
- 成績テキスト
- 獲得賞金
- 父名
- 母の父名
- 近走4走分のテキスト

予想ロジックに直接使う項目は以下。

- 馬番
- 獲得賞金
- 父名
- 母の父名
- 近走4走分のテキスト

近走テキストからは、取得できる範囲で以下を抽出する。

- 着順
- 頭数
- 過去人気
- 着差に相当する括弧内数値
- 距離
- 走破時計
- 通過順

## 共通の近走重み

近走は新しい順に以下の基礎重みを使う。

```text
1走前: 1.00
2走前: 0.72
3走前: 0.52
4走前: 0.36
```

この基礎重みは常にそのまま残す。近走レースのクラスが高い場合だけ、クラス補正値を加算する。

```text
近走重み = 基礎重み + クラス補正値 * クラス補正着順倍率
```

クラス補正は、近走テキストから読み取れるレース格を以下の優先順位で判定する。上にあるものほど大きく加算する。`GIII` は中央重賞の実運用上必要なため、指定順位内の2つ目の `GII` 相当箇所を `GIII` として扱う。

| レース格 | クラス補正値 |
| --- | ---: |
| GI | +0.60 |
| GII | +0.50 |
| Jpn1 | +0.45 |
| GIII | +0.40 |
| OP / Listed | +0.30 |
| 3勝クラス | +0.22 |
| Jpn2 | +0.18 |
| 2勝クラス | +0.12 |
| Jpn3 | +0.08 |
| 1勝クラス以下 | +0.00 |
| グレード不明の重賞 | +0.00 |

クラス補正着順倍率は、加算されるクラス補正値にだけ適用する。基礎重みには影響させない。

```text
8着以下: 0.50
7着以内、または着順不明: 1.00
```

たとえば1走前がGIで3着なら `1.00 + 0.60`、1走前がGIで8着以下なら `1.00 + 0.60 * 0.50` を使う。1勝クラス以下はクラス補正値が `0.00` のため、8着以下でも基礎重みは減らさない。

## 基礎スコア

`score_horse()` が各馬の基礎スコアを計算する。スコアは順位付け用の指数であり、勝率や期待値ではない。

### 獲得賞金

獲得賞金が取得できる場合、円換算した金額に対して以下を加点する。

```text
log10(獲得賞金[円]) * 2.1
```

賞金表記は `億` または `万` を円へ換算する。

### 着順と頭数

近走テキストから着順と頭数が取れる場合、以下を加点する。

```text
max(0, (頭数 + 1 - 着順) / 頭数) * 8.0 * 近走重み
```

3着以内の場合は追加で以下を加点する。

```text
(4 - 着順) * 1.4 * 近走重み
```

### 過去人気

近走テキストから過去人気が取れる場合、以下を加点する。

```text
max(0, 18 - 人気) * 0.12 * 近走重み
```

### 着差

近走テキスト内の括弧付き数値を着差相当として扱う。数値が0以上の場合のみ以下を加点する。

```text
max(0, 2.0 - 着差) * 0.75 * 近走重み
```

### 馬番タイブレーク

最後に馬番で微小補正を加える。

```text
(19 - 馬番) * 0.015
```

これは主に同点時の並びを安定させるための補正である。

## 血統レース条件適性

血統適性は `data/Sire_data.csv` を参照して計算する。対象は父と母の父である。

CSVの主な列:

- `sire_name`
- `surface_axis`
- `distance_m`
- `confidence`
- `source`

`surface_axis` の意味:

```text
-100 = ダート寄り
0    = 中間
100  = 芝寄り
```

レース条件は以下の目標値に変換する。

```text
芝     => 100
ダート => -100
不明   => 0
```

父または母の父の単体適性は以下で計算する。

```text
surface_fit = 1 - min(abs(stallion.surface_axis - race_surface_axis), 200) / 200
distance_fit = 1 - min(abs(stallion.distance_m - race_distance_m), 600) / 600
lineage_fit_score = (surface_fit * 0.55 + distance_fit * 0.45) * 100
```

距離が取れない場合、または該当種牡馬がCSVにない場合、その系統の単体適性は中立値 `50.0` とする。

最終的な血統レース条件適性は以下。

```text
sire_fit_score =
  sire_lineage_fit_score
  + max(0, dam_sire_lineage_fit_score - 50) * 0.35
```

上限は `100.0` とする。母の父は中立値を超えた分だけ加点し、父の評価を下げる方向には使わない。

## 近走由来の追加指数

全馬に近走4走分のテキストがある場合、`calculate_feature_indices()` が追加指数を計算する。欠損値は中立値 `50.0` として扱う。

各指数はレース内の最小値と最大値を使って `0` から `100` に正規化する。全馬が同値、または有効値がない場合は全馬 `50.0` になる。

### 持ちタイム指数

近走テキストから距離と走破時計が取れる場合、速度を計算する。

```text
speed = 距離[m] / 走破秒
```

近走重み付き平均を作り、レース内で0から100に正規化した値を `time_index` とする。高いほど良い。

### 末脚指数

着順品質と通過順からの押し上げで計算する。

```text
finish_quality = (頭数 + 1 - 着順) / 頭数
gain = max(0, 最終通過順 - 着順) / 頭数
closing_raw = finish_quality * 0.65 + gain * 0.35
```

近走重み付き平均を作り、レース内で0から100に正規化した値を `closing_index` とする。高いほど良い。

### 先行力指数

取得できる最初の通過順が前にいるほど高くなるように計算する。

```text
early_position_score =
  1 - (max(最初の通過順, 1) - 1) / (頭数 - 1)
```

近走重み付き平均を作り、レース内で0から100に正規化した値を `pace_index` とする。高いほど良い。

### 総合力指数

基礎スコアをレース内で0から100に正規化した値を `overall_index` とする。

印決定用の総合力ランキングでは以下を使う。

```text
overall_rank_score =
  overall_index
  + time_index * 0.10
  + closing_index * 0.08
  + pace_index * 0.06
  + sire_fit_score * 0.08
```

## 印の決定

`make_picks()` が印を決定する。

### 全馬に近走4走がある場合

全馬が近走4走分のテキストを持つ場合、`make_feature_picks()` を使う。

各印のランキングは以下。

```text
◎: time_index + closing_index が最も高い馬
○: time_index + pace_index が最も高い馬
▲: time_index + closing_index の次点馬
△: overall_rank_score が最も高い馬
☆: sire_fit_score が最も高い馬
```

同じ馬が複数の印候補になる場合は、その印のランキング内で次に高い未選出馬を繰り上げる。候補が尽きた場合は総合力ランキングから未選出馬を補充する。

同点時の並びは以下の順で安定させる。

```text
1. 評価値の降順
2. 馬番昇順
3. 馬名昇順
```

### 近走4走が不足する馬がいる場合

1頭でも近走4走が不足する場合は、fallback方式を使う。

各馬のスコアは以下。

```text
fallback_score = base_score + sire_fit_score * 0.08
```

並び順:

```text
1. fallback_score 降順
2. 馬番昇順
3. 馬名昇順
```

上位から `◎`, `○`, `▲`, `△`, `☆` を順に割り当てる。

## 買い目生成

`bet_sections()` は印が5頭そろった場合のみ買い目を生成する。5頭未満の場合は買い目を生成しない。

現行の生成対象は以下。

### 馬連フォーメーション

```text
◎○ - ○▲△☆
```

同一組み合わせの重複を除外し、合計7点。

### 3連複BOX

```text
◎○▲△☆
```

5頭から3頭を選ぶBOXで、合計10点。

### 3連単フォーメーション

```text
◎○ - ◎○▲ - ◎○▲△☆
```

同一馬の重複を除外し、合計12点。

`穴狙い馬単` は生成しない。

## 的中判定

結果ページ用の買い目結果は `bet_outcomes()` で作る。

まず、確定した1着、2着、3着の馬番を、公開済み予想印に変換する。対象着順に予想印のない馬が含まれる場合、その式別は不的中になる。

式別ごとの判定:

- 馬連: 実際の1着・2着の印が両方あり、その組み合わせが馬連フォーメーション内にある場合のみ的中
- 3連複: 実際の1着・2着・3着の印がすべてあり、その3頭組み合わせが3連複BOX内にある場合のみ的中
- 3連単: 実際の1着・2着・3着の印がすべてあり、その順序が3連単フォーメーション内にある場合のみ的中

払戻額はJRA公式結果から取得した同式別の払戻を使う。的中していても払戻が取得できない場合、払戻額は空欄になる。

結果が未確定の場合は、各式別を `pending` として扱う。

## 現在の制約

- スコアは予測確率ではなく、順位付け用の指数である。
- 現行オッズは印決定に使わない。
- 現行人気順位は表示用であり、印決定には使わない。
- 過去人気は近走テキストから取れる場合のみ基礎スコアに使う。
- 斤量、騎手、性齢、天候、馬場状態は現行スコアに使わない。
- 上がり3Fそのものは安定取得しておらず、末脚指数は着順品質と通過順の押し上げで近似する。
- 近走テキストの構造変化に弱い。
- 血統適性は `data/Sire_data.csv` に登録された父・母の父だけを評価できる。
