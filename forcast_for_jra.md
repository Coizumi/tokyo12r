# JRA Forecast Logic

対象実装: `scripts/jra_site_updater.py`  
確認時点: `25c838a`

## 目的

TOKYO12R by ZIN のJRA予想ロジックは、JRA公式の出走表HTMLを内部処理し、各レースの上位5頭に予想印を付けるための決定的なヒューリスティックです。

機械学習モデル、外部オッズ、天候、馬場状態、血統、調教、コメント、新聞印は現在使用していません。

## 入力データ

`--fetch-official` 実行時に、JRA公式サイトの `JRADB/accessD.html` へPOSTして以下を取得します。

- 開催一覧
- レース一覧
- 各レースの出走表詳細

対象日でレースが見つからない場合は、対象日から最大4日先までを探索し、最初に取得できた開催日を採用します。

各馬について内部的に使う項目は以下です。

- 馬番
- 馬名
- 性齢
- 騎手
- 通算または条件別の成績文字列
- 獲得賞金
- 近走4走分のテキスト

公開JSONおよびHTMLには、馬番、騎手、成績詳細、獲得賞金、過去走詳細、スコアは出しません。公開対象は予想印、馬名、買い目に限定します。

## スコア計算

各馬に対して `score_horse()` が数値スコアを計算します。最終スコアは小数第3位で丸めます。

### 1. 獲得賞金

獲得賞金が取得できる場合、以下を加点します。

```text
log10(獲得賞金[円]) * 2.1
```

賞金表記は `億` または `万` を円に換算します。

### 2. 成績文字列

成績文字列から数字を抽出し、先頭4つを以下として扱います。

```text
勝利数, 2着数, 3着数, 出走数
```

加点式は以下です。

```text
勝利数 * 3.4
+ 2着数 * 2.0
+ 3着数 * 1.2
- 出走数 * 0.16
```

勝利、連対、複勝圏を評価しつつ、出走数が多いだけの馬は少し減点します。

### 3. 近走4走

近走4走は新しい順に以下の重みを掛けます。

```text
1走前: 1.00
2走前: 0.72
3走前: 0.52
4走前: 0.36
```

各走のテキストから、着順、頭数、人気、着差らしき数値を正規表現で拾います。

#### 着順と頭数

着順と頭数が取れる場合、以下を加点します。

```text
max(0, (頭数 + 1 - 着順) / 頭数) * 8.0 * 近走重み
```

上位着順ほど高く、最下位に近いほど低くなります。

さらに3着以内の場合は追加加点します。

```text
(4 - 着順) * 1.4 * 近走重み
```

つまり重み適用前では、1着が `4.2`、2着が `2.8`、3着が `1.4` の追加点です。

#### 人気

人気が取れる場合、以下を加点します。

```text
max(0, 18 - 人気) * 0.12 * 近走重み
```

人気が高いほど加点され、18番人気以下は加点なしです。

#### 着差

括弧内の数値を着差として扱います。数値が0以上の場合のみ、以下を加点します。

```text
max(0, 2.0 - 着差) * 0.75 * 近走重み
```

2.0秒以上の差は加点なしです。負の値は現在無視します。

### 4. 馬番による微小なタイブレーク

最後に馬番でごく小さな補正を入れます。

```text
(19 - 馬番) * 0.015
```

これは実質的に同点時の並びを安定させるための補正です。低い馬番ほどわずかに上になります。

## 印の決定

全馬にスコアを付けた後、以下の順でソートします。

```text
1. スコア降順
2. 馬番昇順
3. 馬名昇順
```

上位5頭に以下の印を割り当てます。

```text
1位: ◎
2位: ○
3位: ▲
4位: △
5位: ☆
```

5頭未満しか印が作れない場合、買い目は生成しません。

## 買い目生成

印が5頭そろった場合、`bet_sections()` が固定フォーメーションで買い目を作ります。

### 馬連フォーメーション

```text
◎○ - ○▲△☆
```

重複を除外し、合計7点です。

```text
◎-○
◎-▲
◎-△
◎-☆
○-▲
○-△
○-☆
```

### 3連複BOX

```text
◎○▲△☆
```

5頭BOXのため、合計10点です。

### 3連単フォーメーション

```text
◎○ - ◎○▲ - ◎○▲△☆
```

同一馬の重複を除外し、合計12点です。

### 穴狙い馬単

```text
☆ - ◎○▲△
```

合計4点です。

## 公開データの削減

内部処理ではスコア、成績、過去走、賞金、馬番、騎手を使いますが、`public_payload()` では以下のみを書き出します。

- 開催日
- 生成時刻
- データソース名
- 会場
- レース番号
- 発走時刻
- レース名
- コース
- 予想印と馬名
- 買い目

これにより、出走表や馬柱の再掲にならないようにしています。

## 現在の制約

- オッズは使っていません。
- 馬場状態、天候、枠順傾向は使っていません。
- 騎手、斤量、性齢は取得しますが、現行スコアには使っていません。
- 血統、調教、厩舎コメント、新聞印は使っていません。
- JRA公式HTMLの構造変更に弱いです。
- 着差の解釈は単純な括弧内数値の抽出です。
- スコアは予測確率ではなく、順位付け用の指数です。
